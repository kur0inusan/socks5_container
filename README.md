# socks5_container

macOS 環境で Docker コンテナを利用して SOCKS5 プロキシサーバーを立てる手順と、その使い方を説明します。

## 使い方

### 1. コンテナの起動

Docker がインストールされて起動していることを確認し、リポジトリのディレクトリで以下のコマンドを実行します。

```bash
docker-compose up -d
```

これにより、バックグラウンドで SOCKS5 プロキシのコンテナが起動します。

### 2. SOCKS5 プロキシの接続情報

コンテナが起動すると、Mac のローカル環境から以下の情報で SOCKS5 プロキシにアクセスできます。

- **IPアドレス**: `127.0.0.1` (または `localhost`)
- **ポート**: `1080`
- **認証**: なし

### 3. Mac での設定方法

Mac 全体の通信をこの SOCKS5 プロキシ経由にする場合は、以下の設定を行います。

1. **システム設定**を開きます。
2. 左側のメニューから **ネットワーク** を選択し、現在接続しているネットワーク（例：Wi-Fi や Ethernet）をクリックします。
3. **詳細...** ボタンをクリックします。
4. **プロキシ** タブを開きます。
5. **SOCKS プロキシ** のスイッチをオンにして有効にします。
6. 以下のように設定します：
   - サーバ: `127.0.0.1`
   - ポート: `1080`
7. **OK** をクリックして設定を保存します。

※ 特定のブラウザ（例：Firefox）などでは、ブラウザ独自の設定画面から SOCKS5 プロキシを指定することも可能です。

### 4. 動作確認

ターミナルから `curl` コマンドを使って、プロキシが正常に動作しているか確認できます。

```bash
curl -x socks5h://127.0.0.1:1080 https://ifconfig.me
```

現在のグローバルIPアドレスが表示されれば成功です。

### 5. コンテナの停止

プロキシサーバーを停止する場合は、以下のコマンドを実行します。

```bash
docker-compose down
```
